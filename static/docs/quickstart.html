APICache Quickstart

APICache is a caching proxy for the Stack Exchange API - but it's more than just a HTTP proxy. Using APICache enables you to save on your API request quota by making use of data requested by other apps in the recent past.

I want to use APICache!

Cool. Find the URL of a server it's being hosted on. For the purposes of documentation, we'll assume it's being hosted on https://cache.example.org/. You will now need:

  • An app registered on Stack Apps to get a request key
  • An existing program you want to adapt to using APICache, or an idea for a new program.

Using APICache is much like using the Stack Exchange API; many of the routes are the same, as are most parameters and response formats. APICache currently supports these routes:

  • /questions/{ids}
  • /answers/{ids}
  • /posts/{ids}
